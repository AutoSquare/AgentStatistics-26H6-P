# AgentStatistics 开发说明

项目概述见 [README.md](../README.md)。

持久化路线：**memory**（`memory` = 记忆快照；`archive` = ZIP 工程包）

## 目录职责

| 路径 | 职责 |
|------|------|
| `Themes/` | DefaultTheme 色板与全局样式 |
| `Resources/` | 图标/背景/字体（ASCII 路径） |
| `Properties/` | Settings.settings / Resource.resx |
| `ViewModel/`、`Services/`、`Model/` | MVVM + Python 桥 |
| `ASPy/`、`ASEnv/` | Python 脚本与运行时 |
| `WebClient/` | Vue + Vite + ECharts 仪表盘源码；构建产物复制到 WPF 输出目录 |

## Codex 统计链路

- Codex 会话定位、解析、缓存、聚合、额度、费用与 CSV 字段语义以 `../CodexScope/generate_codex_data.go` 和 `../CodexScope/app.ts` 为行为真相源。
- 第一阶段不再使用 Go；`ASPy/codex_usage_stats.py` 迁移 CodexScope 核心逻辑并输出前端 JSON。
- 共享聚合逻辑位于 `ASPy/usage_common.py`；Codex / Cursor / Antigravity 均输出 `schemaVersion: 2` 的同款 JSON，前端复用 `WebClient/src/components/AgentDashboard.vue` 与 `charts.ts`。
- 默认会话目录为 `%USERPROFILE%/.codex/sessions`，用户可在界面修改，保存键为 `codexSessionsPath`。
- 增量缓存写入 `%AppData%/AgentStatistics/codex_usage_cache.json`，不写入仓库。
- WPF 主窗口通过 WebView2 加载 `WebClient/dist/index.html`，并通过 WebView message 发送设置、刷新状态与 Codex 统计结果。
- 自动刷新由 WPF 监听 sessions 目录下 `*.jsonl` 文件变化触发；手动刷新按钮保留。
- 隐私边界：只处理 Token、模型、时间戳、会话 id、cwd basename、rate limit、耗时、失败状态等元数据；不读取或导出提示词、助手正文、工具输出或文件内容。

## Cursor 统计链路

- 行为参考 `../token-monitor`：`token-monitor` 通过 tokscale 维护 `%USERPROFILE%/.config/tokscale/cursor-cache/usage*.csv`；本仓库自研 Python 解析器 `ASPy/tokscale_csv.py`，不捆绑 tokscale CLI。
- 统计入口：`ASPy/cursor_usage_stats.py`；C# 包装：`Services/CursorUsageService.cs`。
- 默认缓存目录 `%USERPROFILE%/.config/tokscale/cursor-cache`，界面可改，保存键为 `cursorCachePath`。
- 登录态来源（按优先级，对齐 token-monitor / tokscale 习惯）：
  1. `%USERPROFILE%/.config/tokscale/cursor-credentials.json`（token-monitor / tokscale 已配置账号）
  2. 本机 Cursor 登录缓存 `%APPDATA%/Cursor/User/globalStorage/state.vscdb` 中 `cursorAuth/accessToken`（`ASPy/cursor_discover.py` 自动拼接 `WorkosCursorSessionToken`；**无需保持 Cursor 应用打开**）
- 云端同步：`ASPy/cursor_sync.py` 调用 `https://cursor.com/api/dashboard/export-usage-events-csv?strategy=tokens` 写入 `cursor-cache/usage.csv`；**无需在界面手动粘贴 Token**。
- 额度风险：`ASPy/cursor_limits.py` 探测 `cursor.com/api/usage-summary`，映射为与 Codex 相同的 `risk[]` 面板。
- 增量解析缓存：`%AppData%/AgentStatistics/cursor_usage_cache.json`。
- 自动刷新：WPF 监听 `cursor-cache` 下 `*.csv` 变化；刷新时自动 `--sync`。
- WebView 消息：`cursorData`、`refreshCursor`、`setCursorCachePath`。
- 空数据语义：payload 顶层 `dataStatus`、`sync`、`auth`；前端无 `records` 时显示空状态而非全 0 图表。

### Cursor 故障排查

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| 提示「未检测到 Cursor 登录态」 | 无 tokscale 凭证且 `state.vscdb` 无 `cursorAuth/accessToken` | 在 token-monitor / tokscale 配置账号，或在本机 Cursor 完成一次登录后点刷新 |
| 状态栏「同步失败 HTTP 401/403」 | 登录缓存或 tokscale 凭证过期 | 重新登录 Cursor 账号或更新 tokscale 凭证 |
| 状态栏「同步超时」或 Python `TimeoutError` | Cursor 导出 CSV 响应慢或网络不稳定 | 稍后重试；若 `cursor-cache/usage.csv` 已存在，刷新仍会读取本地缓存 |
| `cursor-cache` 无 `usage*.csv` | 同步未成功或路径错误 | 检查 `%USERPROFILE%\.config\tokscale\cursor-cache`；确认网络可用 |
| CSV 有文件但仍无数据 | 仅表头无数据行，或日期超出默认 30 天窗口 | 确认近期有 AI 调用；切换「历史」视图 |
| 已用 token-monitor 仍无数据 | tokscale 凭证无效或 cursor-cache 为空 | 在 token-monitor 设置中重新登录 Cursor 账号 |

## Antigravity 统计链路

- **CLI 优先**：不依赖 Antigravity IDE。数据根目录优先 `~/.gemini/antigravity-cli`，兼容 `antigravity` / `antigravity-ide` 旧路径（`ASPy/antigravity_paths.py`）。
- 刷新同步：从运行中的 **Antigravity CLI（agy）** 内嵌 `language_server` 经 Connect RPC 拉取轨迹元数据，写入 `antigravity-cache/sessions/*.jsonl` 与 `manifest.json`（`ASPy/antigravity_sync.py`）。进程发现支持 `agy.exe`、CLI 日志端口与无 CSRF 探测（`ASPy/antigravity_connect.py`）。
- Token 历史（离线）：① `antigravity-cache/sessions/*.jsonl`（`ASPy/antigravity_jsonl.py`）；② `brain/<id>/.system_generated/logs/transcript*.jsonl`（`ASPy/antigravity_transcript.py`）。两者在 `antigravity_usage_stats.py` 合并。
- 统计入口：`ASPy/antigravity_usage_stats.py`（支持 `--sync`）；C# 包装：`Services/AntigravityUsageService.cs`（刷新默认带 `--sync`）。
- 默认缓存目录 `%USERPROFILE%/.config/tokscale/antigravity-cache`，保存键为 `antigravityCachePath`；WPF 监听 `sessions/*.jsonl` 变化自动刷新。
- 配额风险：`ASPy/antigravity_quota.py` 复用 `antigravity_connect` 调用 `GetUserStatus`；CLI 未运行时 Token 图可读已有本地数据，风险面板显示等待状态。
- 增量解析缓存：`%AppData%/AgentStatistics/antigravity_usage_cache.json`。
- WebView 消息：`antigravityData`、`refreshAntigravity`、`setAntigravityCachePath`。

### Antigravity 故障排查

| 现象 | 常见原因 | 处理 |
| --- | --- | --- |
| 全 0 / 空状态 | CLI 未运行且本地无 JSONL / transcript | 运行 `agy` 发起对话后点刷新 |
| 状态栏「CLI 未运行」 | 无 `agy` 进程且 CLI 日志端口不可达 | 在终端运行 `agy` 并保持会话 |
| 同步成功但无记录 | 近期无对话，或 RPC 未返回 `generatorMetadata` | 在 CLI 内发起一次对话后重试 |
| 配额面板空 | Connect RPC 不可达 | 与用量同步相同，需 CLI 运行或已有本地缓存 |

## UI/UX 约束

- 任何涉及 WPF 或前端界面结构、组件、图表、导航、空状态、视觉设计或交互改动的任务，先调用 `ui-ux-pro-max`。
- 当前设计系统基线：Data-Dense Dashboard，浅色专业仪表盘，主色 `#1E40AF`，强调色 `#D97706`，图表使用 ECharts。
- 交付前至少检查：文本对比度、焦点态、图表空状态、加载/失败状态、按钮禁用态、375/768/1024/1440 宽度无重叠。
- 图表坐标轴必须启用标签容纳，Token 数值使用 `万`/`亿` 短格式，不能出现坐标轴文字被面板裁掉。
- 历史总览从已有记录开始展示，图表按当前画布宽度自适应聚合为小时、天、月、年；不要把跨天历史图固定显示成 `HH:mm` 小时横轴。
- WPF 主窗口 normal 状态的可用下限为 `1280x820`；启动恢复 `%AppData%/AgentStatistics/user_settings.json` 中的旧窗口宽高时会按 `MinWidth`/`MinHeight` 夹紧，退出保存时同样写入夹紧后的尺寸，避免历史 `1040x680` 等过小布局继续造成 WebView2 内容区被压缩。最大化、还原或尺寸变化后宿主会强制布局刷新，并向前端发送 `dashboardResize`；前端用 `ResizeObserver` 监听主内容区，新增图表时必须接入同一 resize 队列。
- 侧边栏 Tab 使用 `v-if` 仅挂载当前 `AgentDashboard`；切走会卸载组件，切回时新实例 `active` 首帧即为 `true`，非 `immediate` 的 `watch(active)` 不会触发。`AgentDashboard` 须在 `onMounted` 与 `watch(chartView/active, { immediate: true })` 中调用 `tryRenderCharts()`，否则 KPI 有数但 ECharts 空白，需切换时间范围才能恢复。

## memory 模式

- `%AppData%/AgentStatistics/user_settings.json` — 窗口布局（WindowPlacement：宽高、位置、最大化状态）
- `%AppData%/AgentStatistics/session_snapshot.json` — 上次会话业务表 + `uiState.statusText`（状态栏文案）
- `%AppData%/AgentStatistics/WebView2/` — WebView2 用户数据、缓存与运行时状态；`WebClient/dist` 只作为只读前端资源加载，避免安装到 `C:\Program Files` 等受保护目录时写入程序目录。
- 无 `{Ext}` 工程文件菜单
- 契约与接线：Skill 内 `references/ui-state-contract.md`、`references/persistence-wiring.md`

### uiState 扩展键（业务登记）

| 键 | 类型 | 说明 |
|----|------|------|
| `statusText` | string | 模板必选；由 MainWindowViewModel 写入 |

## archive 模式

- 工程包扩展名：`.proj`
- `Assets/Baseline/` — 新建工程种子表
- Export → Driver → Merge → Save/Open（**Ctrl+S**、文件菜单）
- 窗口布局仍写入 `user_settings.json`
- **文件关联**：启动时 `FileAssociationUtilities.TryRegisterOnStartup()` 写入 `HKCU\Software\Classes\.proj`，open 命令 `"exe" "%1"`
- **双击打开**：`App.xaml` 使用 `Startup="Application_Startup"`（**非** `StartupUri`）；`e.Args` 中 `{Ext}` 路径优先于「最近工程」
- Debug/Release 或输出目录变更后须 **再 F5 一次** 刷新关联
- 图标：应用图标使用 `Resources/icons/logo.ico`；前端品牌标识使用 `WebClient/public/logo.png`，来源为 `Resources/icons/logo.png`。

## 构建

1. `setup_python_env.py` 创建 `ASEnv/`
2. `cd WebClient && npm install && npm run build`
3. `dotnet build`
4. `py -3 -c "import sys; sys.path.insert(0,'ASPy'); import unittest; unittest.main(module=None, argv=[''], verbosity=1, exit=True, testRunner=unittest.TextTestRunner(), testLoader=unittest.TestLoader().discover('ASPy', pattern='test_*.py'))"`
5. `smoke_bridge.py` 烟测（需要 Python 环境时）

## 发布 Python 运行时

- `ASEnv/` 是可移植 Python 环境目录，会进入 Release 输出和安装包。
- 发布版 Python 使用便携 Python 3.10.11，放在 `ASEnv/`，应用会优先解析 `ASEnv/python.exe`。
- 发布前必须确认 `ASEnv/` 和 Release 输出中没有 `pyvenv.cfg`、`C:\Users\25705` 或 `Python310` 这类开发机绝对路径。

## 发布 WebView2 运行时

- `ThirdParty/WebView2/MicrosoftEdgeWebView2RuntimeInstallerX64.exe` 是随安装包携带的 Microsoft WebView2 Evergreen Standalone Installer。
- `AgentStatistics安装包/安装包.iss` 会检查 WebView2 Runtime 注册表版本；目标机缺失时自动以 `/silent /install` 静默安装。
- 应用运行时不要求最终用户手动安装 WebView2；如果安装包未能准备运行时，只提示重新运行完整安装包或联系维护人员。

## 版本与安装包

- 软件 EXE 版本在 `AgentStatistics.csproj` 的 `Version`、`AssemblyVersion`、`FileVersion`、`InformationalVersion` 中维护。
- Inno Setup 安装包版本在 `AgentStatistics安装包/安装包.iss` 的 `MyAppVersion` 中维护，并通过 `VersionInfoVersion`、`VersionInfoProductVersion` 写入安装包 EXE 文件属性。
- 发版时保持上述版本号一致，先构建 Release 输出，再确认 `ASEnv/python.exe`、`WebClient/dist/index.html` 和 WebView2 离线安装器都存在，最后编译 `.iss` 安装脚本。
